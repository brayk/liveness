CSC 151 - Liveness Analysis

Write a program that reads a "three address code" program (as seen in MOOC Quiz 6) from standard input and writes to standard output the same program but annotated with "liveness" information.

For simplicity, your program needs only to handle programs using the same conventions as those in MOOC Quiz 6. This means
~ labels will be on their own line and be named "LabelX" where X is a single-digit number,
~ variables will be a single letter,
~ the only operators will be +, *, <, >, and :=.
Additionally all tokens will be space delimited, except for colons after labels.

The last line of the input will be a space delimited, alphabetical list of variables after a # that indicate which variables are live after the last line of the code. Your output should insert a similarly formatted line before and after every instruction (including ifs, but excluding labels and unconditional gotos).

Thus, on input

y := 3
x := y
z := 4 * x
# z

your output should be

#
y := 3
# y
# y
x := y
# x
# x
z := 4 * x
# z

The empty list at the beginning indicates nothing is live coming into the code sequence.
